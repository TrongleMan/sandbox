Part 1 — System Design

The Pluribus Approach

Instead of one generic chat window, build a network of purpose-built AI contexts. Each one knows a domain deeply. A hub routes between them. Memory persists across sessions.

The core insight: the investment is in the context files, not the conversations. Every minute you spend writing a good CLAUDE.md (or system prompt) pays back across every future session. Generic prompts get generic answers. Loaded context gets expert answers.
This approach has 35+ specialized contexts (called personas) organized into layers: a Hub that orchestrates, Agents that run autonomously, Specialists for domain work, Characters that change how you interact, and Tools that invoke and return.
Part 1 — Core Patterns

6 Patterns Worth Stealing

These work at any scale. You don't need 35 personas to benefit from them.

01

The Hub / Orchestrator

One AI context acts as your front door. It reads your current state and routes you to the right place. No re-explaining context every session.

/wm
Opens with your energy level, open tasks, and active projects. You just pick a direction.
02

The Specialist Persona

A context that already knows your resume, your history, and your weak spots. No "tell me about yourself." It remembers.

/career
Stripe systems design on Thursday.
Last time I bombed scalability.
After 10 sessions the advice is calibrated to you specifically, not to "a job seeker."
03

The Character Persona

Personality changes how you engage. A deal-hunter warlock is more fun to consult than "AI Assistant" — and you actually build the habit.

/shopping
About to buy a mechanical keyboard.
Budget $150.
You're more likely to use a tool consistently if it's enjoyable. Character creates relationship.
04

Cross-Tool Integration

AI as connective tissue between your apps. One command routes correctly across Todoist, Calendar, and notes — no micro-decisions.

/capture
[reads Apple Reminders, routes to
Todoist + Calendar automatically]
The system knows your conventions better than you do in the moment.
05

Persistent Memory

Every session writes back to a shared knowledge base. The AI gets smarter about your world over time, not just in the moment.

/wrap
[commits session notes, writes
solution to shared knowledge]
Most AI use is amnesiac. Writeback means compounding returns the longer you use it.
06

The Autonomous Agent

Some contexts aren't for conversation — they run a job and report back. You read it like a briefing.

/gitmaxxing
[scans all repos, flags stale ones,
surfaces cross-project patterns]
You can't hold 35 repos in your head. An agent that audits them for you is a force multiplier.

Part 2 — Tools

Local AI Infrastructure

Run models on your own hardware. Nothing leaves the device.

Ollama
Free Local
One command to pull and run open-source LLMs (Llama, Mistral, Phi, DeepSeek, Gemma) entirely on your machine. No API key, no network calls.
Showcase angle: Pull up a network traffic monitor while running a query. Zero outbound. Powerful demo for anyone with privacy concerns or working with sensitive documents.
LM Studio
Free Local
GUI companion to Ollama. Browse and download models through a polished interface. Has a built-in local OpenAI-compatible API server — any app that talks to ChatGPT can point here instead.
Showcase angle: Show the model browser, pull one, then show the local server endpoint. "Any app that works with ChatGPT works with this, for free."
Open WebUI
Free Local
Drop-in web interface for Ollama. Full ChatGPT experience — conversation history, RAG on uploaded files, multiple models — running entirely locally via Docker.
Showcase angle: Paste in a PDF and ask detailed questions about it. Entirely offline. No data leaves the machine.
Part 2 — Tools

AI Coding Environments

The landscape has exploded. These are the ones actually worth knowing.

Claude Code
Primary
Terminal-first AI coding agent. Reads the whole codebase, writes and edits files, runs commands, works across multiple files simultaneously. Not a chat with copy-paste — an agent with real file access.
Showcase angle: Use the prompts in Part 4. Point it at a messy folder. Watch it restructure everything.
Google Antigravity
New Free Preview
VS Code fork announced November 2025 alongside Gemini 3. Two views: an Editor (like Cursor) and a Manager for orchestrating multiple agents in parallel across workspaces. Supports Gemini 3.1 Pro, Claude Sonnet/Opus, and open-source models.
Showcase angle: Open the Manager view and spawn two agents working on different parts of the same project at the same time. The parallelism is visually striking.
Cursor
The gold-standard AI-native IDE. VS Code fork with Composer (multi-file editing from a description), @ symbols to reference files/docs/web, and shadow workspace for background edits.
Showcase angle: Open a codebase, use Composer to add a feature by description, watch it plan and edit 4 files at once.
opencode
Free
Terminal-based AI coding assistant from the team at SST. Capable coding tool with zero subscription cost if you bring your own API keys or run local models via Ollama.
Showcase angle: Good entry point for developers who live in the terminal and don't want to pay a monthly fee.
VS Code
Free
The free baseline. GitHub Copilot now has a free tier (2000 completions/month). Pair with the Claude Code extension. Most people haven't explored even half of what the free tier offers.
Showcase angle: Show the diff view when AI edits a file, inline suggestions in context, and the chat panel — all without spending anything.
Part 2 — Tools

Voice Input

Eliminate the keyboard as a bottleneck.

Superwhisper
System-wide voice-to-text on Mac. Hold a key, speak, text appears anywhere — terminal, Slack, browser, any app. Runs Whisper locally, no cloud required.
Showcase angle: Open a blank Claude Code prompt, hold the key, describe a whole app verbally, release. The gap between "I had an idea" and "I have code" becomes a few seconds.
Part 2 — Tools

Knowledge & Notes

Your knowledge base as an AI-queryable layer.

Obsidian + CLI
Free
Local markdown vault with an official CLI and community REST API plugin. Point Claude Code directly at your vault directory — it's just files, no plugin required.
Showcase angle: Open your vault in Claude Code. Ask it to find all notes tagged #project and generate a weekly summary. Or restructure your note naming convention across 200 files instantly.
Google NotebookLM
Free
Upload PDFs, Google Docs, URLs, or YouTube links as sources. Ask questions grounded in your sources with citations. The signature feature: Audio Overview — a two-host podcast-style discussion of your material generated in ~30 seconds.
Showcase angle: Drop in a dense research paper or a long document, generate the Audio Overview, and play it. Watch someone's reaction. Also powerful for: interview prep, briefing docs, book summaries.
Part 2 — Tools

Image, Video & Audio Generation

The generation stack — all from within Google's ecosystem.

Gemini — Image Generation
Free Tier
Imagen-powered image generation built into the Gemini app. Text-to-image for product mockups, concept art, social graphics, and more.
Showcase angle: Give it a specific product brief or character description. Compare speed and cost vs. commissioning a designer for a rough concept.
Nano Banana / Nano Banana Pro
New
Google's text-based image editor (Gemini 2.5 Flash / Gemini 3 Pro). Edit images by describing changes in plain English: "remove the background," "make her hair red," "add rain," "put text on the shirt." Nano Banana Pro is best-in-class for rendering legible text inside images — something most models still fail at.
Showcase angle: Take any photo, make three chained edits in plain English. Show consistency across edits — characters and scene stay coherent.
Gemini — Video Generation (Veo)
Gemini Advanced
Prompt-to-video generation via Veo, available in Gemini Advanced. Describe a scene, get a short video clip.
Showcase angle: Describe a 5-second clip. Play the result. Point out what it would have cost to shoot or animate even a rough version of that. The trajectory matters more than the current quality.
Google MusicFX — Song Generation
Free
Text-to-music via Google AI Test Kitchen. Describe genre, mood, tempo, and instrumentation. Generates loopable tracks in seconds.
Showcase angle: Describe a very specific vibe: "lo-fi hip hop, rainy day, slow BPM, vinyl crackle." Play it. The precision of the output vs. the simplicity of the input is the reveal.
Google Stitch
New
Prompt-to-UI. Describe an app screen in plain language, Stitch generates working HTML/CSS. Built for rapid prototyping and wireframing.
Showcase angle: Describe a settings screen for a mobile app. Generate it. Copy the output, open in browser. "From description to working UI in under a minute."
Part 3 — Tutorial Demos

Live Demos to Run

Concrete, repeatable demos that land well with a non-technical audience.

Demo 1: Messy Folder → Structured Data

Create a folder of 20-30 inconsistently named files (scan_final_FINAL.pdf, IMG_4291.jpg, johns birthday - draft 3.docx). Then run a single Claude Code prompt. Everyone has a folder like this.

1
Create ~/Desktop/demo-mess/ with poorly named files across several categories
2
Open Claude Code: "Infer what each file is, rename to YYYY-MM-DD_category_description.ext, and generate a CSV index with: original name, new name, category, confidence."
3
Show the renamed folder and the generated CSV. Total elapsed: under 60 seconds.

Demo 2: Xcode — App from a Comment

Open a blank SwiftUI project in Xcode 16. With Claude Code or GitHub Copilot active, start with a single comment and watch it fill in.

1
New Xcode project → SwiftUI → blank ContentView.swift
2
Type: // Tip calculator: bill input, tip slider 10–25%, party size, per-person breakdown, animated total
3
Let AI fill in the implementation. Run in simulator. A working feature from one description.

Demo 3: NotebookLM Audio Overview

The fastest "wait, what?" reaction in the AI demo catalog.

1
Drop a dense PDF (research paper, long article, anything they'd never actually read) into NotebookLM
2
Click Generate Audio Overview. Wait ~30 seconds.
3
Play the two-host podcast discussion of that exact document. Watch the reaction.

Demo 4: Nano Banana Chained Edits

Shows the difference between generation and editing — and why consistency across edits matters.

1
Open Nano Banana with any photo (a portrait, a product shot, a landscape)
2
Edit 1: "Remove the background, replace with a coffee shop interior." Edit 2: "Add warm golden hour lighting." Edit 3: "Add a coffee cup on the table next to them."
3
Show the consistency — same person, same clothes, same camera angle — across all three edits.
